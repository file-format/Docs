## Che cos'è un file VSSX?

I file con estensione .vssx stanno disegnando stencil creati con [Microsoft Visio](https://products.office.com/en/visio/flowchart-software) 2013 e versioni successive. Il formato di file VSSX può essere aperto con Visio 2013 e versioni successive. I file Visio sono noti per la rappresentazione di una varietà di elementi di disegno come raccolta di forme, connettori, diagrammi di flusso, layout di rete, diagrammi UML, diagrammi software, modelli di database, mappatura di oggetti e altre informazioni simili. Microsoft Visio offre anche la possibilità di convertire i file Visio in diversi formati di file come [PNG](/it/image/png/), [BMP](/it/image/bmp/), [PDF](/it/pdf/) e altri. È disponibile sia per Windows che per Mac OS.

## Formato file VSSX

Il formato di file VSSX si basa sul formato OpenOffice adottato da Microsoft dal 2007. Si basa sull'archivio [ZIP](/it/compression/zip/) che rappresenta il formato di file generale basato sulle specifiche del formato di file XML. L'equivalente in formato file binario di VSSX era VSS, supportato fino a Visio 2007. È possibile visualizzare il contenuto del formato file VSSX sostituendo la sua estensione con .ZIP e aprirlo in qualsiasi formato di file di archiviazione come WinZIP.

I file VSDX sono basati su Open Packaging Conventions e XML e gli sviluppatori possono trarre vantaggio da questo formato imparando a lavorare con questi file a livello di codice. Il formato eredita molte delle stesse strutture XML delle sue parti dal formato di file di disegno XML di Visio (.vdx). L'interoperabilità con i file di Visio è notevolmente aumentata poiché il software di terze parti può manipolare i file di Visio a livello di formato file.

Alcuni altri tipi di file che comprendono il formato di file di Visio 2013 includono:

* .vsdm (disegno abilitato alle macro di Visio)
* .vssx (stencil di Visio)
* .vssm (stencil con abilitazione macro di Visio)
* .vstx (modello di Visio)
* .vstm (modello abilitato per le macro di Visio)

Ogni file di Visio è definito come pacchetto che contiene altri file o parti. Una parte del pacchetto può essere un file XML, un'immagine o anche una soluzione VBA. Le parti all'interno del pacchetto possono essere suddivise in parti "documento" e "relazione".
