## Какво е UDL файл?
Файлът с разширение .udl се нарича Microsoft Universal Data Link файл; определяне на атрибути на връзката; използвани от приложенията на Windows за установяване на връзка с базата данни. UDL файлът съдържа низа за връзка за източник на данни OLE DB; с потребителско име и парола и основни свойства на низа за връзка. За да се избегне директното указване на свойствата на ръка в низ за връзка, диалоговият прозорец Свойства на връзката към данни може да се използва за запазване на информация за връзка в .udl файл като алтернатива.

## UDL файлов формат
По принцип UDL (Universal Data Link) файл е прост текстов файл, който се състои от низ за свързване към база данни с конкретни атрибути или свойства. След като UDL файлът бъде създаден, той може да бъде тестван с помощта на SQL Server Management Studio за проверка на свързаността.

### Свойства на низа за връзка
Следните свойства могат да бъдат зададени в UDL, за да се гарантира правилното свързване:

- **Име на сървъра**: местоположението на сървъра, където се намира базата данни, до която искате да получите достъп.
- **Опции за удостоверяване**
- **Удостоверяване на Windows**: Удостоверяване на SQL Server с помощта на идентификационните данни на Windows акаунта на текущо влезлия потребител.
- **Удостоверяване на SQL Server**: Удостоверяване чрез ИД за влизане и парола.
- **Active Directory - Integrated**: Интегрирано удостоверяване с идентичност на Azure Active Directory; може да се използва и за удостоверяване на Windows към SQL Server.
- **Active Directory - Парола**: Потребителско име и удостоверяване с парола с идентичност на Azure Active Directory.
- **Active Directory - Универсален с MFA поддръжка**: Интерактивно удостоверяване с идентичност на Azure Active Directory.
- **Active Directory - Service Principal**: Удостоверяване с принципал на услугата Azure Active Directory.
- **Server SPN**: Ако използвате доверена връзка, можете да посочите основно име на услуга (SPN) за сървъра.
- **Потребителско име**: Въведете потребителския идентификатор, който да използвате за удостоверяване, когато влизате в източника на данни.
- **Парола**: Въведете паролата, която да използвате за удостоверяване, когато влизате в източника на данни.
- **Празна парола**: Когато е отметнато, позволява на посочения доставчик да използва празна парола в низа за връзка.
- **Разрешаване на запазване на парола**: Позволява паролата да бъде запазена с низа за връзка.
- **Използвайте силно криптиране за данни**: данните, които се предават през връзката, ще бъдат криптирани.
- **Сертификат на доверен сървър**: Сертификатът на сървъра ще бъде валидиран.
- **База данни**: Име на база данни, до която искате да получите достъп.
- **Прикачете файл на база данни като име на база данни**: Указва името на основния файл за база данни с възможност за прикачване.
